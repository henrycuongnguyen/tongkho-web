# Scout Report: V2 Listing Implementation - Current State

**Analysis Complete** | 2026-02-11 10:38 AM

## Executive Summary

The v2 listing system has a **clean, modular architecture** with proper separation of concerns. Zero results are currently handled **only at UI level** with a simple empty state message. No fallback/relaxation logic exists.

## Key Architecture Findings

### 1. **Search Flow Path** (Linear, No Fallbacks)
```
URL → Parser → Location Resolution → ES Query Building → ES Execution → Results Display
```

- **URL Parser** (`listing-url-parser.ts`): Converts `/mua-ban/ha-noi/gia-tu-1-ty-den-2-ty?property_types=12,13` into `PropertySearchFilters`
- **Location Resolution** (`[...slug].astro` lines 98-157): Converts location slugs to ES-compatible IDs (using `nId` strings, NOT integer IDs)
- **Query Builder** (`query-builder.ts`): Creates ES bool query with must/filter conditions
- **ES Execution** (`property-search-service.ts`): Fetches from `real_estate` index, returns `{total, hits, took}`
- **Display** (`listing-grid.astro`): Conditionally renders 3-col grid OR empty state message

**Key Advantage:** No additional API calls needed! Fallback can reuse already-fetched data.

### 5. **Multi-Location Support Already Built**
- **Primary:** From URL path (e.g., `/mua-ban/ha-noi`)
- **Additional:** Via `addresses` query param (comma-separated slugs)
- **Smart expansion:** Only expands province to all districts if NO specific districts provided
- **Example:** `/mua-ban/ha-noi?addresses=ba-dinh,dong-da` searches both districts only

### 6. **ElasticSearch ID System (CRITICAL)**
- **Database PK:** Integer `id` (e.g., `123`)
- **ElasticSearch ID:** String `nId` (e.g., `"VN-HN"`)
- **Query builder MUST use:** String IDs for location filters!
- **Common bug:** Using integer IDs results in zero results (already fixed in current code)

### 7. **Search Query Structure**
```typescript
// Generated by query-builder.ts
{
  query: {
    bool: {
      must: [
        { term: { transaction_type: 1 } },
        { terms: { property_type_id: [12, 13] } },
        { terms: { district_id: ["VN-HN-HBT", "VN-HN-HKM"] } },  // Uses nId!
        { range: { price: { gte: 1B, lte: 2B } } },
        { range: { area: { gte: 50 } } },
        { term: { bedrooms: 3 } }
      ],
      filter: [{ term: { aactive: true } }]
    }
  },
  from: 0,
  size: 24,
  sort: [{ created_on: "desc" }]
}
```

### 8. **Error Handling (Silent Failures)**
- ES request errors → logged server-side, returns `{total: 0, hits: [], took: 0}`
- SEO metadata failures → gracefully skipped (continue without)
- Side block failures → logged and skipped
- **No user messaging** when search unavailable
